# Initiatives

Initiatives group related specs under a shared goal. Every spec's `initiative:` field references an entry here.

| ID | Slug | Description |
|---|---|---|
| INI-001 | sdlc-throughput | Reduce the friction in the spec → implementation → review → merge loop without losing the rigor that catches real defects. Originates from observed pain in high-gear's SPEC-042 execution. |
| INI-002 | sdlc-onboarding | Significantly simplify the path from "I want to use this framework" to "I have shipped my first spec." User-flagged priority. |
| INI-003 | sdlc-artifact-completeness | Port high-gear's artifact-shape improvements upstream (evidence field, spec_followups, gap capture, etc.). |
| INI-004 | sdlc-multi-agent | Port high-gear's multi-agent coherence patterns (three role-specific entry points, worktree isolation, monorepo dep graph). |

New initiatives are added by editing this file as part of the spec that introduces them.
